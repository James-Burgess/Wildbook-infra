Configuration

Environment-specific configuration lives in conf/base/ with local overrides in conf/local/.

Data Catalog

conf/base/catalog.yml defines all datasets used by the pipeline:

raw_images:
  type: pandas.CSVDataset
  filepath: data/01_raw/images.csv

preprocessed_images:
  type: pandas.CSVDataset
  filepath: data/02_intermediate/preprocessed_images.csv

features:
  type: pandas.CSVDataset
  filepath: data/03_primary/features.csv

report:
  type: pandas.CSVDataset
  filepath: data/04_report/report.csv

Each dataset maps to a CSV file in the data/ directory. Kedro automatically loads and saves these as pandas DataFrames.

Parameters

conf/base/parameters.yml stores pipeline parameters:

images_path: data/01_raw/images.csv

Parameters are accessible in nodes via params:<key> syntax in the pipeline definition.

Local Overrides

Place environment-specific overrides in conf/local/. These files are gitignored and should contain:

  • credentials.yml — API keys, database passwords, secrets
  • Any catalog or parameter overrides for your local environment

Kedro Settings

src/wildbook_pipeline/settings.py controls Kedro's runtime behavior:

  • CONFIG_LOADER_ARGS — sets base_env to base and default_run_env to local
  • CONF_SOURCE — defaults to conf
  • Hook registration, session class, and catalog class can be customized here

Pipeline Registration

src/wildbook_pipeline/pipeline_registry.py discovers and registers all pipelines. The __default__ pipeline is the sum of all discovered pipelines.

To add a new pipeline, create a module under src/wildbook_pipeline/pipelines/ with a create_pipeline() function. Kedro auto-discovers it.
