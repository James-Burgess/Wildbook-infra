Kedro Testing Patterns

Automated testing is critical for maintaining code quality as your Kedro project grows. Kedro recommends pytest as the primary testing framework, with support for fixtures, mocks, and coverage reporting.

Why Automated Testing Matters

As a project grows, new code increasingly depends on existing code. Manual testing (running the pipeline and checking results) becomes impractical — it's time-consuming and often misses regressions. Automated tests run comprehensive suites in seconds, catching breaking changes during development rather than in production.

Setting Up Tests

Install Test Dependencies

# Add to pyproject.toml
[project.optional-dependencies]
dev = [
    "pytest-cov",
    "pytest-mock",
    "pytest",
]

# Install project + test deps
pip install -e ".[dev]"

Alternately, install individually:

pip install -e .          # editable install of your project
pip install pytest
pip install pytest-cov
pip install pytest-mock

Test Directory Structure

Mirror your src/<package_name> layout under tests/. Name files test_<file_being_tested>.py:

src/
└── wildbook_pipeline/
    └── pipelines/
        └── example/
            ├── __init__.py
            └── nodes.py

tests/
└── pipelines/
    └── example/
        └── test_nodes.py

Writing Tests

Testing Nodes (Unit Tests)

Kedro nodes are pure Python functions, making them easy to test in isolation:

# tests/pipelines/example/test_nodes.py
import pandas as pd
import pytest
from wildbook_pipeline.pipelines.example.nodes import (
    load_raw_images,
    preprocess_images,
    extract_features,
    generate_report,
)

class TestLoadRawImages:
    def test_load_raw_images_returns_dataframe(self, tmp_path):
        """Test that load_raw_images reads a CSV and returns a DataFrame."""
        csv_path = tmp_path / "images.csv"
        csv_path.write_text("filename\nimg001.jpg\nimg002.jpg\n")

        result = load_raw_images(str(csv_path))
        assert isinstance(result, pd.DataFrame)
        assert len(result) == 2
        assert list(result.columns) == ["filename"]

class TestPreprocessImages:
    def test_drops_null_rows(self):
        """Test that null rows are removed."""
        df = pd.DataFrame({"filename": ["img001.jpg", None, "img003.jpg"]})
        result = preprocess_images(df)
        assert len(result) == 2
        assert result["filename"].isna().sum() == 0

    def test_strips_whitespace(self):
        """Test that leading/trailing whitespace is removed."""
        df = pd.DataFrame({"filename": ["  img001.jpg  ", "img002.jpg"]})
        result = preprocess_images(df)
        assert result["filename"].tolist() == ["img001.jpg", "img002.jpg"]

class TestExtractFeatures:
    def test_produces_feature_vectors(self):
        """Test that features are computed correctly."""
        df = pd.DataFrame({"filename": ["img001.jpg", "img002.jpg"]})
        result = extract_features(df)
        assert "feature_vector" in result.columns
        assert len(result) == 2
        for vec in result["feature_vector"]:
            assert isinstance(vec, list)
            assert len(vec) == 2

class TestGenerateReport:
    def test_generates_report_strings(self):
        """Test that report column is created."""
        df = pd.DataFrame({
            "filename": ["img001.jpg", "img002.jpg"],
            "feature_vector": [[1, 2], [3, 4]],
        })
        result = generate_report(df)
        assert "report" in result.columns
        assert all(":" in r for r in result["report"])

Testing Nodes with Fixtures

Use fixtures to share test data across multiple tests:

import pytest
import pandas as pd

@pytest.fixture
def sample_images_df():
    """Reusable fixture: a small DataFrame of image filenames."""
    return pd.DataFrame({
        "filename": ["  whale_001.jpg  ", "whale_002.jpg", None, "shark_003.jpg"],
        "width": [1024, 2048, 800, 1920],
    })

class TestPreprocessing:
    def test_cleans_data_with_fixture(self, sample_images_df):
        result = preprocess_images(sample_images_df)
        assert result["filename"].str.startswith(" ").sum() == 0  # no leading spaces
        assert result["filename"].isna().sum() == 0                # no nulls

Testing Pipeline Definitions

Verify that pipelines are well-formed and have correct inputs/outputs:

from wildbook_pipeline.pipelines.example import create_pipeline

class TestExamplePipeline:
    def test_pipeline_has_four_nodes(self):
        pipeline = create_pipeline()
        assert len(pipeline.nodes) == 4

    def test_pipeline_input_is_params(self):
        pipeline = create_pipeline()
        # Pipeline's overall input should include params reference
        inputs = pipeline.inputs()
        assert "params:images_path" in inputs

    def test_pipeline_output_is_report(self):
        pipeline = create_pipeline()
        outputs = pipeline.outputs()
        assert "report" in outputs

Integration Test: KedroSession

To test that the full pipeline runs end-to-end, use KedroSession:

from pathlib import Path
from kedro.framework.session import KedroSession
from kedro.framework.startup import bootstrap_project

class TestKedroRun:
    def test_kedro_run_completes(self):
        bootstrap_project(Path.cwd())
        with KedroSession.create(project_path=Path.cwd()) as session:
            result = session.run(pipeline_name="example")
            assert result is not None

Testing with Mocks

Mock external dependencies or slow operations:

from unittest.mock import patch, MagicMock

def test_load_with_mock(mocker):
    """Use pytest-mock to mock pd.read_csv."""
    mock_read = mocker.patch("pandas.read_csv")
    mock_read.return_value = pd.DataFrame({"filename": ["test.jpg"]})

    from wildbook_pipeline.pipelines.example.nodes import load_raw_images
    result = load_raw_images("does/not/exist.csv")
    assert len(result) == 1
    mock_read.assert_called_once_with("does/not/exist.csv")

Running Tests

# Run all tests
pytest

# Run specific file
pytest tests/pipelines/example/test_nodes.py

# Run specific test
pytest tests/pipelines/example/test_nodes.py::TestPreprocessImages::test_drops_null_rows

# Verbose output
pytest -v

# Stop on first failure
pytest -x

Test Coverage with pytest-cov

Add coverage configuration to pyproject.toml:

[tool.pytest.ini_options]
addopts = """
--cov-report term-missing \
--cov src/wildbook_pipeline -ra"""

Run with coverage:

pytest

Output includes a summary of statement coverage per file, with the --term-missing option showing uncovered line numbers:

Name                                         Stmts   Miss  Cover   Missing
--------------------------------------------------------------------------
src/wildbook_pipeline/__init__.py                 3      0   100%
src/wildbook_pipeline/pipelines/example/nodes.py 20      3    85%   12-14
...

Testing Naming Conventions

  • Test classes: Test<Name>
  • Test functions/methods: test_<description>
  • Test files: test_<file_being_tested>.py
  • Be descriptive: test_drops_null_rows is better than test_passes

CI Integration

In CI (GitHub Actions, CircleCI, etc.), Kedro's Rich logging auto-detects the "dumb" terminal and strips formatting. To adjust log message width in CI:

export COLUMNS=120 LINES=25

For a typical CI workflow, your test step might be:

# .github/workflows/test.yml
- name: Run tests
  run: |
    pip install -e ".[dev]"
    pytest --cov=src/wildbook_pipeline --cov-report=xml

Wildbook Pipeline Test Configuration

Our project uses make test which invokes kedro test (a wrapper around pytest). Tests are run inside the Docker container during CI:

# From Makefile
test:
    $(KEDRO) test
