Production

The production stack uses Apache Airflow with CeleryExecutor for scheduled, distributed pipeline execution.

Deployment

# Build the Airflow image
make prod-build

# Initialize the database and create admin user
make prod-init

# Start the stack
make prod-up

Open http://localhost:8080 and log in with admin / admin.

TargetDescription

DAG

The wildbook_pipeline DAG runs the example pipeline on a @daily schedule. It uses a single BashOperator task that executes:

cd /opt/airflow/wildbook-pipeline && kedro run --pipeline example

The DAG is defined in dags/wildbook_pipeline.py and auto-discovered by Airflow on startup.

Architecture

The stack uses CeleryExecutor for distributed task execution:

  • PostgreSQL stores DAG metadata, run history, and task state
  • Redis acts as the Celery message broker and result backend
  • Scheduler monitors DAGs and queues tasks
  • Worker picks up tasks from Redis and executes them
  • Triggerer handles deferrable (async) operators
  • Webserver provides the Airflow UI
