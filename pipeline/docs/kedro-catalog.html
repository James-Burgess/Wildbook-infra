Kedro Data Catalog

The Data Catalog is Kedro's registry of all data sources available for the project. It is specified via YAML configuration (catalog.yml) and maps node input/output names to dataset classes.

Important: From Kedro 0.19.0, datasets are no longer in the core package. Install them from kedro-datasets. From version 2.0.0, all dataset names changed to lowercase "s": CSVDataSet → CSVDataset.

Basic Configuration

The catalog.yml file registers datasets by name, type, and storage location:

companies:
  type: pandas.CSVDataset
  filepath: data/01_raw/companies.csv

reviews:
  type: pandas.CSVDataset
  filepath: data/01_raw/reviews.csv

shuttles:
  type: pandas.ExcelDataset
  filepath: data/01_raw/shuttles.xlsx
  load_args:
    engine: openpyxl

Each entry requires three things: a dataset name (top-level key), a type (the dataset class), and a filepath (storage location). Additional parameters depend on the specific dataset implementation.

Dataset Type

Kedro supports many formats: CSV, Excel, Parquet, Feather, HDF5, JSON, pickled objects, SQL tables, SQL queries, and more. Full documentation at kedro-datasets.

Common types include:

  • pandas.CSVDataset
  • pandas.ExcelDataset
  • pandas.ParquetDataset
  • json.JSONDataset
  • matplotlib.MatplotlibWriter
  • pillow.ImageDataset
  • snowflake.SnowparkTableDataset

Filepath Protocols

Kedro uses fsspec to support multiple storage backends. If no protocol is given, the local filesystem (file://) is assumed:

ProtocolExample
Localfile:///path/to/data (or relative path)
Amazon S3s3://my-bucket/path/to/data
S3 Compatible (MinIO)s3://my-bucket/path/to/data
Google Cloud Storagegcs://bucket-name/path
Azure Blob / ADLS Gen2abfs://container/path
Hadoop HDFShdfs://user@server:port/path
HTTP(S)http:// or https:// (read-only)

Additional filesystems (SSH, FTP, WebHDFS) are supported via fsspec.

Load, Save, and Filesystem Arguments

load_args / save_args

Passed directly to the underlying library's read/write methods:

cars:
  type: pandas.CSVDataset
  filepath: data/01_raw/cars.csv
  load_args:
    sep: ','
  save_args:
    index: False
    date_format: '%Y-%m-%d %H:%M'
    decimal: .

fs_args

Configures filesystem interaction. The open_args_load and open_args_save sub-keys control how files are opened:

test_dataset:
  type: pandas.CSVDataset
  filepath: gcs://bucket/data.csv
  fs_args:
    project: my-gcp-project          # passed to GCSFileSystem
    open_args_load:
      mode: "rb"
      encoding: "utf-8"
    open_args_save:
      mode: "a"                       # append mode

Credentials

Store credentials in conf/local/credentials.yml (never commit):

# conf/local/credentials.yml
dev_s3:
  client_kwargs:
    aws_access_key_id: YOUR_KEY
    aws_secret_access_key: YOUR_SECRET

Reference them in catalog.yml:

motorbikes:
  type: pandas.CSVDataset
  filepath: s3://your_bucket/data/motorbikes.csv
  credentials: dev_s3
  load_args:
    sep: ','

Kedro reads credentials before instantiating the DataCatalog and passes them to dataset constructors.

Dataset Versioning

Enable by setting versioned: True:

cars:
  type: pandas.CSVDataset
  filepath: data/01_raw/cars.csv
  versioned: True

Each pipeline run creates a timestamped version at filepath/<YYYY-MM-DDThh.mm.ss.sssZ>/cars.csv.

By default, kedro run loads the latest version. Load a specific version:

kedro run --load-versions=cars:2024-01-15T10.30.00.000Z

List all available versions programmatically:

dataset = catalog._datasets["cars"]
versions = dataset.list_versions(full_path=False)
# ['2024-01-15T10.30.00.000Z', '2024-01-14T09.15.00.000Z', ...]

Versioning is supported by all datasets that extend kedro.io.AbstractVersionedDataset. Note that HTTP(S) cannot be combined with versioning.

Environment-Specific Catalogs

Use conf/base/catalog.yml for production and conf/local/catalog.yml for development overrides:

# conf/base/catalog.yml — production
cars:
  filepath: s3://my_bucket/cars.csv
  type: pandas.CSVDataset

# conf/local/catalog.yml — development override
cars:
  filepath: data/01_raw/cars.csv
  type: pandas.CSVDataset

Kedro loads conf/base/ first, then merges conf/local/ on top. The local definition takes precedence.

Parameters Datasets

Parameters are defined in conf/base/parameters.yml and accessed in nodes via params:<key>:

# conf/base/parameters.yml
images_path: data/01_raw/images.csv
train_fraction: 0.8
random_state: 42
# In pipeline definition
node(
    func=load_raw_images,
    inputs="params:images_path",
    outputs="raw_images",
    name="load_node",
)
